# ביורוקרטיה סיטי וואש — веб-страница для генерации документов сотрудников

Отдельная веб-страница, оформленная как чат в WhatsApp. Спрашивает, какой
документ нужен, задаёт несколько вопросов и отдаёт заполненный PDF. Никакого
мессенджера и аккаунта Meta не требуется — просто ссылка, которую открываешь
в браузере.

## Какие документы умеет делать

1. 📝 חוזה עבודה – מנהל/ת תחנה — трудовой договор для станционных менеджеров
2. 📝 חוזה עבודה – עובד/ת — трудовой договор для рядовых работников (с вопросами
   про должность и график: фиксированный или сменный)
3. 📄 מכתב פיטורים — письмо об увольнении
4. 📋 הזמנה לשימוע — приглашение на слушание перед увольнением
5. ✅ אישור העסקה — справка о трудоустройстве
6. 🦺 הצהרת בטיחות — декларация об инструктаже по технике безопасности
7. 🚗 הודעה על אירוע/נזק לביטוח — извещение в страховую о повреждении (например,
   клиент задел щётку/машину). День недели в форме подставляется автоматически
   по дате события — отдельно не спрашивается.

Чтобы добавить ещё один тип документа — пришли свой .docx-шаблон, я токенизирую
его так же, как остальные, и добавлю в список.

**Автоматическая связка:** при создании любого из двух договоров (пункты 1 и 2)
к нему всегда автоматически прикрепляется הצהרת בטיחות (пункт 6) — получаешь
один PDF-файл с обоими документами. Имя и ת.ז. берутся из ответов на договор,
дата подписания инструктажа = дата начала работы. Настраивается в `web_app.py`
(словарь `BUNDLES`).

## Файлы

- `web_app.py` — веб-сервер (FastAPI): отдаёт страницу чата и API для генерации PDF,
  содержит `DOCUMENT_REGISTRY` со списком всех типов документов и их шаблонов
- `chat_ui.html` — сама страница чата (HTML/CSS/JS, оформлена под WhatsApp),
  содержит `DOC_TYPES` с вопросами для каждого документа
- `contract_filler.py` — заполняет .docx-шаблон и конвертирует в PDF
  (`fill_document` — универсально для любого шаблона, `fill_contract` — с
  дополнительной логикой оклад/почасовая для трудового договора)
- `employment_contract_template_ABT.docx`, `employment_contract_template_worker.docx`,
  `template_piturim.docx`, `template_shimua.docx`, `template_ishur_haaskaa.docx`,
  `template_betichut.docx`, `template_incident_notice.docx` — токенизированные шаблоны
- `Dockerfile` — образ с Python + LibreOffice (нужен для конвертации в PDF)
- `requirements.txt`
- `whatsapp_bot_app.py`, `.env.example` — вариант с реальным WhatsApp (Meta Cloud API),
  на будущее, если решишь вернуться к нему — не обязателен для веб-версии

## Как посмотреть локально (для проверки, не обязательно)

```
pip install -r requirements.txt
uvicorn web_app:app --reload --port 8000
```
Открыть http://localhost:8000 в браузере.

## Деплой — чтобы страница была доступна всегда по ссылке

Нужен любой хостинг, который запускает Docker-контейнеры. Например Render.com
(есть бесплатный тариф). Render требует репозиторий на GitHub — прямой загрузки
файлов без Git у них нет.

1. Создать репозиторий на GitHub и загрузить в него файлы (можно через веб-интерфейс
   GitHub — "Add file → Upload files", без командной строки).
2. Render → New → Web Service → подключить этот репозиторий → **Environment: Docker**
   (подхватит `Dockerfile` автоматически, ничего дополнительно настраивать не нужно —
   переменные окружения из `.env.example` для этой версии не требуются).
3. После деплоя Render даст постоянную ссылку вида `https://имя.onrender.com` —
   это и есть твой "чат". Можно открывать с телефона или компьютера, добавить
   на главный экран телефона как ярлык.

## Как это работает для тебя

1. Открываешь ссылку — бот спрашивает, какой документ нужен (5 кнопок)
2. Отвечаешь по очереди на вопросы (разные для каждого типа документа)
3. Бот показывает сводку — нажимаешь "✅ כן, צור מסמך"
4. Через пару секунд появляется кнопка "⬇️ הורד/י PDF" — скачиваешь готовый документ
5. Чтобы сделать ещё один — обновляешь страницу

## Важно перед реальным использованием

- Каждый сеанс — отдельный, данные хранятся только в браузере на время диалога,
  на сервере не сохраняются.
- Ссылку на страницу стоит держать при себе — она не защищена паролем, кто угодно
  с ссылкой сможет сгенерировать договор. Если это критично, можно добавить простой
  пароль на страницу — скажи, добавлю.
- Договор — это юридический документ. Рекомендуется хотя бы один раз показать
  готовый шаблон и процесс юристу/бухгалтеру перед массовым использованием.
- В шаблоне остались пустыми поля для подписи и даты подписания (заполняются от руки).
